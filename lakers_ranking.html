Linear Ordering of NBA Players from the Los Angeles Lakers Based on Selected Statistical Features

1. Summary

This project performs a linear ordering of Los Angeles Lakers players based on selected basketball statistics. Nine diagnostic features were analyzed, including points, assists, shooting efficiency, and turnovers. The goal was to create a player ranking and evaluate the consistency of results using different data normalization methods. Destimulant variables were transformed into stimulants, and three normalization techniques were applied: unitarization, standardization, and ratio transformation. For each method, a synthetic development index (Qi) was calculated, resulting in three different rankings. The results were compared and conclusions were drawn regarding player classification.

2. Keywords

Linear ordering, normalization, standardization, unitarization, ratio transformation, synthetic indicator, multivariate analysis, object ranking, stimulants, destimulants, nominants, player ranking.

3. Introduction

The idea behind multivariate comparative analysis is to construct a synthetic indicator that reflects the evaluation criterion of comparable multidimensional objects. The construction of this indicator linear ordering depends on many factors, including the nature of the diagnostic variables, their measurement scales, the weight system, and the normalization and aggregation methods.

def preprocess_data(df, all_features):
    df = df[df['Yrs'] > 1]

    df = df[df['MP'].notna()]
    df[all_features] = df[all_features].fillna(df[all_features].median())
    
    per_minute_features = ['ORB', 'AST', 'STL', 'BLK', 'TOV', 'PTS']
    for feature in per_minute_features:
        df[feature] = df[feature] / df['MP']

    return df

  • Outliers and Their Handling

The Interquartile Range (IQR) method[3] was applied. Based on the quartiles Q1 (25th percentile) and Q3 (75th percentile), the IQR, which is a measure of data dispersion, was calculated. The boundaries for outlying observations were computed as follows:

  • Lower boundary = Q1 - 1.5 IQR

  • Upper boundary = Q3 + 1.5 IQR

Observations that fall outside this range were considered outliers.

Below is the identification of outliers for the BLK feature:

def impute_outliers_IQR(df, features):
    
    for feature in features:
        q1 = df[feature].quantile(0.25)
        q3 = df[feature].quantile(0.75)
        IQR = q3 - q1

        lower_bound = q1 - 1.5 * IQR
        upper_bound = q3 + 1.5 * IQR
        mean_val = df[feature].mean()

        df.loc[df[feature] > upper_bound, feature] = mean_val
        df.loc[df[feature] < lower_bound, feature] = mean_val

    return df

  • Division into Stimuli, Detractors, and Nominals

When performing the ranking, features are divided into 3 groups based on preferences:

  • stimuli, where higher values of this feature classify the object as "better",
  • detractors, where higher values of this feature classify the object as "worse",
  • nominals, where the normal level gives the object a "higher" position, and deviations from this level result in a "lower" position.

In research, it is usually required that all features be stimuli. Detractors can then be transformed into stimuli using a differential formula:

$$ \Large x_{ij}^S = max_{i}(x_{ij}) - x_{ij}^D $$

where:

  • i object number
  • j variable number
  • xij the value being studied (i.e., the j-th feature of the i-th object)

6. Normalization of Variables

When creating a ranking of objects based on quantitative features, it is necessary to normalize the variables.

Normalization refers to "standardizing or standardizing feature values from the perspective of a specific criterion." The main goal of this type of transformation is to satisfy the additivity requirement, meaning bringing features with different units of measurement into a comparable form. There are numerous methods for normalizing quantitative features[4]. Within a given normalization formula, there are simple formulas that standardize the characteristics of variables.

Selected methods of normalization:

  • Unit normalization
  • Standardization
  • Ratio transformation

  1. Unit normalization (MUZ method)

    In this method, the variances and means of the variables (features) remain different, but all of these features will take values within the convenient computation interval [0, 1]. The transformed variables behave as stimuli (values closer to 1 indicate the best values of the selected diagnostic variables):

    $$ \Large z_{ij} = \frac{x_{ij} - \min_i(x_{ij})}{\max_i(x_{ij}) - \min_i(x_{ij})} $$

    Thus, the smallest value in the dataset is subtracted from the value being studied (i.e., the j-th feature of the i-th object, e.g., shooting efficiency), and the result is divided by the range, i.e., the difference between the maximum and minimum values.

In[13]:
def min_max_scaling(df, features):
    df_norm = df.copy()
    df_norm[features] = (df_norm[features] - df_norm[features].min()) / (df_norm[features].max() - df_norm[features].min())
    return df_norm

  1. Standardization

    Standardization transforms the features so that they have a variance equal to 1 and a mean equal to 0. Some numbers will change from positive to negative (which is possible, as shown by the numerator of the fraction: this occurs when a value is smaller than the mean).

    All values xij are transformed in this way, provided they are stimuli: $$ \Large z_{ij} = \frac{x_{ij} - \overline{x}_j}{S(x_j)} \quad i = 1, 2, \dots, n; \quad j = 1, 2, \dots, m $$

    Here, xj represents the arithmetic mean of the j-th variable, and S(xj) denotes the standard deviation of that same j-th variable.

def standardization(df, features):

  1. Ratio Transformation with the Mean as the Basis for Normalization

    In this method, each feature value is compared to the arithmetic mean of that feature. As a result, we obtain values expressed as a proportion relative to the mean. This allows us to assess how much a given value deviates from the "typical" level. The formula is as follows:

    $$ \Large z_{ij} = \frac{x_{ij}}{\overline{x}_j} $$

    Here, xj represents the arithmetic mean of the j-th variable, and xij represents the j-th feature of the i-th object.

def ratio_transformation(df, features):

7. Construction of the Synthetic Indicator

The calculation of the aggregate variable Qi using the formula below serves as the basis for constructing a ranking of players based on their statistics. The indicator is built using the method of summing normalized variables. The results obtained depend on the selected normalization formulas.

In our analysis, each feature was assigned an equal weight of 1, ensuring that all features are considered equally important.

$$ \Large Q_i = \sum_{j=1}^{k} a_j z_{ij} $$

where:

  • i object number
  • j variable number
  • aj weight for the given variable
  • zij normalized value of variable j for object i
  • Qi synthetic indicator for the given object

The higher the value of the synthetic variable Qi, the higher the object ranks.

In[16]:
def compute_synthetic_index(df, weights=None):
    if weights is None:
        weights = np.ones(df.shape[1]) / df.shape[1]
    return df.dot(weights)

10. Conclusions:

  • Ranking Consistency

Four players who appeared in the TOP 10 across all three methods are Flynn Robinson, Devontae Cacok, Earl Tatum, and Connie Hawkins. This indicates that these players maintain high positions regardless of the transformation method used, which suggests their stable performance across all normalization techniques.

  • Impact of the Normalization Method

The analysis shows that changing the data transformation method (e.g., standardization, unitarization, ratio transformation) can affect the ranking outcomes of players, even though the structure of the data remains unchanged.

  1. Standardization and unitarization yield more similar results

For 23 players who held identical positions in the rankings after normalization and unitarization, we observe that these two transformation methods produce similar outcomes for most players. This may be because both methods largely preserve the proportions between the values of the statistics.

  1. Ratio Transformation Causes Greater Divergence

Ratio transformation is more sensitive to changes in the proportions between statistics. Flynn Robinson is the only player who maintained the same position in all three rankings, which shows that the ratio method has a greater impact on changing player positions in the ranking.

  • Limitations of the Analysis

    The ranking is based solely on the available numerical data and does not take into account contextual factors, such as the player's role in the team or injuries. Additionally, the selection of specific diagnostic variables and their weight in the analysis can significantly influence the final result.

11. Performance Evaluation

In summary, the analysis not only allowed for the creation of rankings that order players based on their performance but also identified the differences resulting from the choice of a specific data transformation method.

def min_max_scaling(df, features):
    return (df[features] - df[features].min()) / (df[features].max() - df[features].min())

def standardization(df, features):
    return (df[features] - df[features].mean()) / df[features].std()

def ratio_transformation(df, features):
    return df[features].div(df[features].mean())

def compute_synthetic_index(df, weights=None):
    if weights is None:
        weights = np.ones(df.shape[1]) / df.shape[1]
    return df.dot(weights)

def preprocess_data(df, all_features):
    df = df[df['Yrs'] > 1]

    df = df[df['MP'].notna()]
    df[all_features] = df[all_features].fillna(df[all_features].median())
    #df.loc[:, all_features] = df[all_features].fillna(df[all_features].median())
    
    per_minute_features = ['ORB', 'AST', 'STL', 'BLK', 'TOV', 'PTS']
    for feature in per_minute_features:
        df[feature] = df[feature] / df['MP']
        #df.loc[:, feature] = df[feature] / df['MP']

    return df
    
def impute_outliers_IQR(df, features):
    
    for feature in features:
        q1 = df[feature].quantile(0.25)
        q3 = df[feature].quantile(0.75)
        IQR = q3 - q1

        lower_bound = q1 - 1.5 * IQR
        upper_bound = q3 + 1.5 * IQR
        mean_val = df[feature].mean()

        df.loc[df[feature] > upper_bound, feature] = mean_val
        df.loc[df[feature] < lower_bound, feature] = mean_val

    return df

def transform_destimulants(df, destimulants):
    for feature in destimulants:
        df[feature] = df[feature].max() - df[feature]
    return df
    
def plot_histograms(df, features, output_dir):
    for feature in features:
        plt.figure(figsize=(6, 4))
        sns.histplot(df[feature], kde=True, bins=20)
        plt.title(f'Histogram: {feature}')
        plt.show()
        plt.close()
        
def generate_rankings(df, all_features, method_dict):
    for name, method in method_dict.items():
        df_norm = method(df.copy(), all_features)
        df["Synthetic_Index"] = compute_synthetic_index(df_norm[all_features])
        ranking = df[["Player", "Synthetic_Index"]].sort_values(by="Synthetic_Index", ascending=False)
        output_file = f"ranking_lakers_{name}.csv"
        ranking.to_csv(output_file, index=False)
        print(f"Top 5 players ({name}):")
        print(ranking.head(5), "\n")
        
def main():
    filepath = "players1.csv"
    df = load_lakers_data(filepath)
    df.head()

        
    features = classify_features()
    all_features = features['stimulants'] + features['destimulants']

    df = preprocess_data(df, all_features)
    df = impute_outliers_IQR(df, all_features)
    df = transform_destimulants(df, features['destimulants'])

    plot_histograms(df, all_features, output_dir="plots/histograms")

    transformations = {
        'unit normalization': min_max_scaling,
        'standardization': standardization,
        'ratio transformation': ratio_transformation
    }

    generate_rankings(df, all_features, transformations)


if __name__ == "__main__":
    main()
